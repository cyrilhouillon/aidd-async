# 03 - Probe

Compose one batched round of pointed questions sized for an async exchange.

## Input

The idea so far, its altitude, and the settled points.

## Output

The comment body only, a one-line framing of where the idea stands, then the questions grouped by theme and numbered continuously; `05-post` adds the skeleton and resume block.

## Process

1. **Map the live threads.** List every open thread that still forks the build, not only the biggest one. An async round-trip is expensive, so one round covers them all. See the batching guidance in [probing.md](../references/probing.md).
2. **Probe at altitude.** Ask at the idea's grain, functional or technical. Never drop a level finer into how-to that belongs to planning.
3. **Challenge.** Surface unstated assumptions and walk the limit cases that would change what gets built. Draw tactics from [probing.md](../references/probing.md).
4. **Compose.** Group the questions by theme, number them continuously across themes so the author can answer by number, and name each fork plainly. Never repeat a question the thread already answers, and never pad with a checklist.

## Test

- The questions are pointed, sit at the idea's altitude, none is already answered in the thread, they are grouped by theme and numbered, and every open fork is named.
